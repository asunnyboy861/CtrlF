### 模块 1: ScreenshotCaptureManager (截图保存)

#### 需求描述
实现真正的截图保存功能：截取当前相机预览 + 高亮叠加层的合成图像，保存到系统相册。该功能为 Pro 专属。

#### 设计原则
- 单一职责：只负责截图合成与保存
- 复用现有 `CameraPreviewView` 和 `HighlightOverlayView` 的渲染逻辑
- Pro 限制：通过 `isProUser` 参数控制，非 Pro 用户触发 Paywall

#### 接口设计
```swift
import SwiftUI

/// 截图捕获管理器
/// 职责：将相机预览与高亮叠加层合成为 UIImage 并保存到相册
class ScreenshotCaptureManager {
    
    /// 捕获指定视图的截图
    /// - Parameters:
    ///   - view: 要截图的 SwiftUI View（传入 ContentView 的根 ZStack）
    ///   - isProUser: 是否为 Pro 用户
    ///   - completion: 完成回调，返回成功/失败状态
    static func capture(
        view: some View,
        isProUser: Bool,
        completion: @escaping (Result<Void, CaptureError>) -> Void
    )
    
    enum CaptureError: Error {
        case notProUser           // 非 Pro 用户
        case renderingFailed      // 视图渲染失败
        case saveFailed(Error)    // 相册保存失败
    }
}
```

#### 实现规则
1. 使用 `ImageRenderer` (iOS 16+) 将 SwiftUI View 渲染为 `UIImage`
2. 请求 `NSPhotoLibraryAddUsageDescription` 权限（已配置）
3. 保存成功后触发触觉反馈
4. 非 Pro 用户调用时立即返回 `.failure(.notProUser)`

#### 代码示例
```swift
import SwiftUI

class ScreenshotCaptureManager {
    enum CaptureError: Error, LocalizedError {
        case notProUser
        case renderingFailed
        case saveFailed(Error)
        
        var errorDescription: String? {
            switch self {
            case .notProUser: return "Screenshot capture is a Pro feature"
            case .renderingFailed: return "Failed to render screenshot"
            case .saveFailed(let error): return error.localizedDescription
            }
        }
    }
    
    static func capture(
        view: some View,
        isProUser: Bool,
        completion: @escaping (Result<Void, CaptureError>) -> Void
    ) {
        guard isProUser else {
            completion(.failure(.notProUser))
            return
        }
        
        let renderer = ImageRenderer(content: view)
        renderer.scale = UIScreen.main.scale
        
        guard let uiImage = renderer.uiImage else {
            completion(.failure(.renderingFailed))
            return
        }
        
        UIImageWriteToSavedPhotosAlbum(uiImage, nil, nil, nil)
        completion(.success(()))
    }
}
```

#### 测试用例
```swift
// Test 1: Pro 用户截图成功
let mockView = Color.red.frame(width: 100, height: 100)
ScreenshotCaptureManager.capture(view: mockView, isProUser: true) { result in
    assert(result.isSuccess)
}

// Test 2: 非 Pro 用户截图失败
ScreenshotCaptureManager.capture(view: mockView, isProUser: false) { result in
    assert(result.isFailure)
    assert(result.error == .notProUser)
}
```

#### 废弃清理
- 废弃 `ContentView.captureScreenshot()` 中的旧实现（仅调用 `recordSearch`）
- 替换为调用 `ScreenshotCaptureManager.capture()`
- 保留 `recordSearch` 调用在搜索提交时执行

### 模块 2: HighlightColorManager (高亮颜色管理)

#### 需求描述
将 `SettingsView` 中选择的高亮颜色持久化，并实时同步到 `HighlightOverlayView`。

#### 设计原则
- 单一数据源：使用 `@AppStorage` 持久化用户选择
- 继承现有：复用 `SettingsView` 中的 `colorNameToColor` 方法
- 最小改动：通过 `AppConstants` 扩展提供颜色映射，避免硬编码

#### 接口设计
```swift
import SwiftUI

/// 高亮颜色管理器
/// 职责：管理用户选择的高亮颜色，提供颜色解析与持久化
class HighlightColorManager: ObservableObject {
    @AppStorage("highlightColorName") var colorName: String = "Yellow"
    
    /// 当前选择的 SwiftUI Color
    var color: Color { HighlightColorManager.color(for: colorName) }
    
    /// 将颜色名称解析为 Color
    static func color(for name: String) -> Color
    
    /// 所有可用颜色选项
    static let availableColors = ["Yellow", "Green", "Blue", "Orange", "Pink"]
}
```

#### 实现规则
1. 使用 `@AppStorage("highlightColorName")` 持久化
2. `HighlightOverlayView` 通过 `@Environment` 或参数接收颜色
3. 默认颜色为 Yellow（与现有行为一致）
4. 颜色变化时自动触发视图刷新

#### 代码示例
```swift
import SwiftUI

class HighlightColorManager: ObservableObject {
    @AppStorage("highlightColorName") var colorName: String = "Yellow"
    
    var color: Color {
        Self.color(for: colorName)
    }
    
    static func color(for name: String) -> Color {
        switch name {
        case "Yellow": return .yellow
        case "Green": return .green
        case "Blue": return .blue
        case "Orange": return .orange
        case "Pink": return .pink
        default: return .yellow
        }
    }
    
    static let availableColors = ["Yellow", "Green", "Blue", "Orange", "Pink"]
}

// HighlightOverlayView 修改
struct HighlightOverlayView: View {
    let matchedItems: [MatchedTextItem]
    @StateObject private var colorManager = HighlightColorManager()
    
    var body: some View {
        Canvas { context, size in
            for item in matchedItems {
                let rect = convertBoundingBox(item.boundingBox, in: size)
                let highlightRect = Path(roundedRect: rect, cornerRadius: 4)
                let color = colorManager.color
                
                context.fill(highlightRect, with: .color(color.opacity(0.35)))
                context.stroke(highlightRect, with: .color(color), lineWidth: 2)
            }
        }
        .allowsHitTesting(false)
    }
}
```

#### 测试用例
```swift
// Test 1: 默认颜色为 Yellow

assert(manager.color == .yellow)

// Test 2: 修改颜色后同步更新
manager.colorName = "Green"
assert(manager.color == .green)

// Test 3: 无效颜色名回退到 Yellow
assert(HighlightColorManager.color(for: "Invalid") == .yellow)
```

#### 废弃清理
- 删除 `SettingsView` 中的 `@State private var highlightColor`（改为使用 `HighlightColorManager`）
- 删除 `SettingsView` 中的 `colorNameToColor` 方法（迁移到 `HighlightColorManager`）
- `SettingsView` 通过 `@StateObject` 持有 `HighlightColorManager`

### 模块 3: AudioFeedbackManager (音频反馈)

#### 需求描述
当用户搜索到匹配结果时，播放简短的音频提示音。该功能为 Pro 专属，可通过设置开关控制。

#### 设计原则
- 单一职责：只负责音频播放控制
- 复用现有：使用系统 `AVSpeechSynthesizer` 或 `AudioServicesPlaySystemSound`
- Pro 限制：非 Pro 用户开关禁用或隐藏

#### 接口设计
```swift
import AVFoundation

/// 音频反馈管理器
/// 职责：控制搜索匹配时的音频反馈
class AudioFeedbackManager: ObservableObject {
    @AppStorage("audioFeedbackEnabled") var isEnabled: Bool = false
    
    /// 播放匹配成功提示音
    func playMatchSound()
    
    /// 播放搜索提交提示音
    func playSearchSound()
}
```

#### 实现规则
1. 使用 `AudioServicesPlaySystemSound(1103)` 播放系统提示音（轻量，无需音频文件）
2. 仅在 `isEnabled == true` 时播放
3. 在 `TextRecognitionEngine` 匹配到结果时触发
4. Pro 限制：设置页面中开关仅在 Pro 用户时可用

#### 代码示例
```swift

            audioManager?.playMatchSound()
        }
        // ...
    }
}
```

### 模块 4: LowLightAutoFlashManager (低光自动手电筒)

#### 需求描述
当环境光线不足时，自动开启手电筒辅助照明。该功能为 Pro 专属，可通过设置开关控制。

#### 设计原则
- 单一职责：只负责亮度检测与手电筒控制
- 复用现有：复用 `FlashlightController`
- Pro 限制：设置页面中开关仅在 Pro 用户时可用

#### 接口设计
```swift
import AVFoundation

/// 低光自动手电筒管理器
/// 职责：检测环境亮度并自动控制手电筒
class LowLightAutoFlashManager: ObservableObject {
    @AppStorage("autoFlashlightEnabled") var isEnabled: Bool = false
    
    /// 检查当前帧亮度并决定是否开启手电筒
    /// - Parameter sampleBuffer: 相机采集的 CMSampleBuffer
    func evaluateLighting(sampleBuffer: CMSampleBuffer)
    
    /// 手动重置状态
    func reset()
}
```

#### 实现规则
1. 从 `CMSampleBuffer` 提取亮度信息（通过 `AVCaptureDevice` 的曝光值近似）
2. 当亮度低于阈值时自动开启手电筒
3. 当亮度恢复时自动关闭手电筒
4. 仅在 `isEnabled == true` 且为 Pro 用户时工作
5. 避免频繁切换：添加状态防抖（2 秒间隔）

#### 代码示例
```swift
import AVFoundation

class LowLightAutoFlashManager: ObservableObject {
    @AppStorage("autoFlashlightEnabled") var isEnabled: Bool = false
    
    private var lastToggleTime: Date = .distantPast
    private let toggleInterval: TimeInterval = 2.0
    private var isCurrentlyOn: Bool = false
    
    func evaluateLighting(sampleBuffer: CMSampleBuffer) {
        guard isEnabled else { return }
        guard Date().timeIntervalSince(lastToggleTime) > toggleInterval else { return }
        
        // 通过 capture device 获取当前曝光值判断亮度
        guard let device = AVCaptureDevice.default(for: .video) else { return }
        let exposureValue = device.exposureTargetOffset
        let isLowLight = exposureValue > 1.5 // 曝光补偿高 = 光线暗
        
        if isLowLight && !isCurrentlyOn {
            FlashlightController.toggle(true)
            isCurrentlyOn = true
            lastToggleTime = Date()
        } else if !isLowLight && isCurrentlyOn {
            FlashlightController.toggle(false)
            isCurrentlyOn = false
            lastToggleTime = Date()
        }
    }
    
    func reset() {
        if isCurrentlyOn {
            FlashlightController.toggle(false)
            isCurrentlyOn = false
        }
    }
}
```

#### 测试用例
```swift
// Test 1: 启用且低光时开启手电筒
let manager = LowLightAutoFlashManager()
manager.isEnabled = true
// 模拟低光环境
manager.evaluateLighting(mockLowLightSample)
assert(FlashlightController.isOn == true)

// Test 2: 禁用时无操作
manager.isEnabled = false
manager.evaluateLighting(mockLowLightSample)
assert(FlashlightController.isOn == false)
```

#### 废弃清理
- 无需废弃代码（新增功能）
- `SettingsView` 中的 `@State private var autoFlashlight` 替换为 `@StateObject var lowLightManager`

### 模块 5: ProFeatureGate (Pro 功能限制统一入口)

#### 需求描述
统一所有 Pro 功能的访问控制，避免散落在各处的 `isProUser` 判断逻辑。

#### 设计原则
- 单一职责：集中管理 Pro 功能权限判断
- 继承现有：复用 `StoreManager.isProUser`
- 最小改动：通过扩展或包装器实现，不修改现有业务逻辑

#### 接口设计
```swift
/// Pro 功能门禁
/// 职责：统一管理哪些功能需要 Pro 权限
enum ProFeature {
    case screenshotCapture    // 截图保存
    case audioFeedback        // 音频反馈
    case autoFlashlight       // 自动手电筒
    case customHighlightColor // 自定义高亮颜色
    case unlimitedSearches    // 无限搜索
    case searchHistory        // 搜索历史
}

extension StoreManager {
    /// 检查指定功能是否需要 Pro 权限
    func requiresPro(_ feature: ProFeature) -> Bool
    
    /// 检查当前用户是否有权限使用指定功能
    func canAccess(_ feature: ProFeature) -> Bool
}
```

#### 实现规则
1. 所有 Pro 功能枚举集中定义
2. `StoreManager` 扩展提供统一的权限检查
3. UI 层调用 `canAccess()` 判断是否显示/启用功能
4. 非 Pro 用户尝试使用 Pro 功能时触发 Paywall

#### 代码示例
```swift
enum ProFeature: CaseIterable {
    case screenshotCapture
    case audioFeedback
    case autoFlashlight
    case customHighlightColor
    case unlimitedSearches
    case searchHistory
}

extension StoreManager {
    func requiresPro(_ feature: ProFeature) -> Bool {
        switch feature {
        case .screenshotCapture, .audioFeedback, .autoFlashlight,
             .customHighlightColor, .unlimitedSearches, .searchHistory:
            return true
        }
    }
    
    func canAccess(_ feature: ProFeature) -> Bool {
        guard requiresPro(feature) else { return true }
        return isProUser
    }
}
```

#### 测试用例
```swift
// Test 1: 免费用户无法使用 Pro 功能
let store = StoreManager()
store.isProUser = false
assert(store.canAccess(.screenshotCapture) == false)

// Test 2: Pro 用户可以使用所有功能
store.isProUser = true
assert(store.canAccess(.screenshotCapture) == true)
assert(store.canAccess(.audioFeedback) == true)
```

#### 废弃清理
- 逐步替换各 View 中分散的 `isProUser` 判断为 `storeManager.canAccess(.feature)`
- 保留 `isProUser` 属性本身（作为底层状态）
