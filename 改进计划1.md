# Improvement Plan 1

## 1. Onboarding Experience

**Need**: Users need immediate guidance on first launch. The app shows a camera view with a search bar but no explanation of what CtrlF does or how to use it.

**Generation Rules**:
- Create OnboardingView with 3 swipeable pages
- Page 1: "Point your camera at any text" with book illustration
- Page 2: "Type the word you're looking for" with search illustration
- Page 3: "See results highlighted instantly" with highlight illustration
- Show only on first launch using UserDefaults
- Use .fullScreenCover presentation
- Inherit yellow accent color from app design system

**Verification**: First launch shows onboarding, subsequent launches skip it. UserDefaults key "hasCompletedOnboarding" persists.

## 2. Search Counter Logic Fix

**Need**: Current implementation only records searches when "Capture" button is tapped, but the actual search happens in real-time as user types. Free tier limit should decrement when user submits a search query, not when they capture.

**Generation Rules**:
- Move search recording to when user taps "Search" on keyboard (onSubmit)
- Add onSubmit modifier to search TextField
- Keep Capture button for screenshot saving only
- Update remaining count display immediately after each search

**Verification**: Type text → tap Search on keyboard → counter decrements. Capture button no longer affects search count.

## 3. OCR Language Setting Connection

**Need**: SettingsView has OCR Language picker but it doesn't affect TextRecognitionEngine's recognition languages.

**Generation Rules**:
- Add @AppStorage for ocrLanguage preference
- Pass language setting to TextRecognitionEngine
- Map display language names to BCP-47 codes
- Update recognitionLanguages in processFrame

**Verification**: Change language in Settings → OCR recognizes text in selected language.

## 4. Keyboard Submit Action

**Need**: Search bar TextField should have a "Search" return key type and submit action for better UX.

**Generation Rules**:
- Add .submitLabel(.search) to TextField
- Add onSubmit handler that records the search
- Dismiss keyboard on submit

**Verification**: Tap "Search" on keyboard → search recorded → keyboard dismissed.

## 5. Safe Area Padding for Search Bar

**Need**: Search bar may overlap with status bar on devices with notch/Dynamic Island.

**Generation Rules**:
- Add top padding that respects safe area
- Use GeometryReader or safeAreaInset modifier
- Ensure search bar is always fully visible

**Verification**: On iPhone 15 Pro simulator, search bar is fully below status bar.
