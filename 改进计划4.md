## 问题背景

### 用户反馈
"进入页面后自动默认的是live页面，我点击Library进入Library支持图片，但是我再点击Live后长时间没有反应，很迟钝很久又直接跳出来让我升级到Pro莫名其妙"

### 问题总结

| # | 问题 | 严重程度 | 根因 |
|---|------|----------|------|
| 1 | Live ↔ Library 切换迟钝（3-10秒） | 🔴 严重 | `switchToCameraMode()` 调用 `cameraManager.startSession()`，`startRunning()` 是同步阻塞操作 |
| 2 | 切换模式时莫名弹出 Paywall | 🔴 严重 | `updateSearchQuery()` 中有 `!canSearch` 检查，搜索词不为空+免费次数用完=弹 Paywall |
| 3 | 点击 Library 后需手动再选照片 | 🟡 中 | `handlePhotoImport()` 只切换模式，不自动弹出照片选择器 |
| 4 | 免费用户可访问 Search History（Pro功能） | 🟡 中 | `onHistory` 回调直接设置 `showHistory = true`，无 Pro 检查 |
| 5 | Photo 模式高亮位置偏移 | 🟡 中 | `HighlightOverlayView` 使用全屏尺寸，但 `scaledToFit` 照片周围有 letterbox |
| 6 | Settings 中 Pro 功能无拦截 | 🟡 中 | Highlight Color / Sound Feedback / Auto Flashlight 对免费用户完全开放 |

## 修复方案

### 修复 1: 模式切换卡顿 — 相机 Session 持续运行

#### 需求描述
消除 Live ↔ Library 模式切换时的长时间等待，实现即时切换。

#### 根因分析
```
切换前:
  switchToPhotoMode() → cameraManager.stopSession()   ← 停止相机（1秒）
  switchToCameraMode() → cameraManager.startSession()  ← 重启相机（3-10秒！）

AVCaptureSession.startRunning() 是同步阻塞调用
在模拟器上（无真机摄像头）尤其慢
```

#### 解决方案
相机 session 保持持续运行，不在模式切换时 stop/start。Photo 模式下仅跳过帧处理。

#### 修改文件
- `SearchViewModel.swift` — `switchToPhotoMode()` / `switchToCameraMode()`
- `TextRecognitionEngine.swift` — 新增 `isCameraMode` 属性

#### 代码变更

**SearchViewModel.swift — 模式切换不再操作相机**
```swift
// 修改前:
func switchToPhotoMode() {
    searchMode = .photo
    cameraManager.stopSession()     // ← 删除：不再停止相机
}

func switchToCameraMode() {
    searchMode = .camera
    selectedPhoto = nil
    selectedPhotoItem = nil
    textEngine.matchedItems = []
    cameraManager.startSession()    // ← 删除：不再重启相机
}

// 修改后:
func switchToPhotoMode() {
    searchMode = .photo
    textEngine.isCameraMode = false  // ← 新增：跳过帧处理
}

func switchToCameraMode() {
    searchMode = .camera
    textEngine.isCameraMode = true   // ← 新增：恢复帧处理
    if let photo = selectedPhoto {
        selectedPhoto = nil
        selectedPhotoItem = nil
    }
}
```

**TextRecognitionEngine.swift — Photo 模式跳过帧处理**
```swift
// 新增属性:
var isCameraMode: Bool = true

// 修改 processFrame():
func processFrame(_ pixelBuffer: CVImageBuffer) {
    guard isCameraMode else { return }  // ← 新增：Photo 模式直接跳过
    processingFrameCount += 1
    guard processingFrameCount % frameSkipInterval == 0 else { return }
    // ... 原有逻辑
}
```

### 修复 2: 意外 Paywall 弹出 — 移除搜索查询中的额度拦截

#### 需求描述
防止模式切换时不必要地触发 Paywall，只在用户主动提交搜索时检查额度。

#### 根因分析
```
switchToCameraMode() 设置 textEngine.isCameraMode = true
→ 相机帧开始处理
→ searchText 不为空时触发 updateSearchQuery()
→ 旧逻辑: !canSearch && !searchText.isEmpty → showPaywall = true
→ 免费次数用完的用户：每次切换模式都弹 Paywall！
```

#### 解决方案
`updateSearchQuery()` 只负责将搜索词传给 textEngine，不检查额度。额度检查只在 `submitSearch()` 时执行。

#### 修改文件
- `SearchViewModel.swift` — `updateSearchQuery()`

#### 代码变更
```swift
// 修改前:
func updateSearchQuery() {
    if !canSearch && !searchText.isEmpty {
        showPaywall = true    // ← 删除：不应在查询更新时弹 Paywall
        return
    }
    textEngine.updateSearchQuery(searchText)
}

// 修改后:
func updateSearchQuery() {
    textEngine.updateSearchQuery(searchText)
}
```

#### 额度检查保留位置
```swift
// submitSearch() 中已有完整检查:
private func submitSearch() {
    guard viewModel.canSearch else {
        viewModel.showPaywall = true  // ← 唯一正确的触发点
        return
    }
    viewModel.recordSearch(context: modelContext)
}
```

### 修复 5: Photo 模式高亮坐标对齐

#### 需求描述
照片模式下，当照片宽高比与屏幕不一致时（letterbox），高亮框位置偏移。

#### 根因分析
```
HighlightOverlayView 使用全屏尺寸计算 boundingBox 坐标
但 Image.scaledToFit() 在照片周围留白（letterbox）
导致高亮框与实际文字位置不对齐
```

#### 解决方案
计算照片实际渲染区域，让 HighlightOverlayView 仅覆盖照片区域。

#### 修改文件
- `ContentView.swift` — `photoModeView`

#### 代码变更
```swift
// 修改前:
GeometryReader { geometry in
    Image(uiImage: photo)
        .resizable()
        .scaledToFit()
        .frame(width: geometry.size.width, height: geometry.size.height)
        .position(x: geometry.size.width / 2, y: geometry.size.height / 2)
}
HighlightOverlayView(matchedItems: viewModel.textEngine.matchedItems)
    .ignoresSafeArea()  // ← 全屏尺寸，letterbox 导致偏移

// 修改后:
GeometryReader { geometry in
    let imageAspect = photo.size.width / photo.size.height
    let viewAspect = geometry.size.width / geometry.size.height
    let fitWidth = imageAspect > viewAspect ? geometry.size.width : geometry.size.height * imageAspect
    let fitHeight = imageAspect > viewAspect ? geometry.size.width / imageAspect : geometry.size.height

    ZStack {
        Image(uiImage: photo)
            .resizable()
            .scaledToFit()
            .frame(width: geometry.size.width, height: geometry.size.height)

        HighlightOverlayView(matchedItems: viewModel.textEngine.matchedItems)
            .frame(width: fitWidth, height: fitHeight)  // ← 精确对齐照片区域
            .position(x: geometry.size.width / 2, y: geometry.size.height / 2)
    }
}
```

### 修复 6: Settings Pro 功能拦截

#### 需求描述
Settings 中的 Highlight Color、Sound Feedback、Auto Flashlight 是 Pro 功能，免费用户应看到锁定标识并触发 Paywall。

#### 修改文件
- `SettingsView.swift` — 新增 `proGateRow()` 和 `proToggle()` helper

#### 代码变更

**新增 Pro 功能拦截 helper 方法**
```swift
@ViewBuilder
private func proGateRow<Content: View>(icon: String, title: String, isPro: Bool, currentValue: String, @ViewBuilder proContent: () -> Content) -> some View {
    if isPro {
        proContent()
    } else {
        Button {
            showPaywall = true
        } label: {
            HStack {
                Image(systemName: icon).foregroundStyle(.secondary).frame(width: 24)
                Text(title).foregroundStyle(.primary)
                Spacer()
                Text(currentValue).foregroundStyle(.secondary)
                Image(systemName: "lock.fill").font(.caption2).foregroundStyle(.yellow)
            }
        }
    }
}

@ViewBuilder
private func proToggle(icon: String, title: String, isOn: Binding<Bool>, isPro: Bool) -> some View {
    if isPro {
        Toggle(isOn: isOn) { Label(title, systemImage: icon) }
    } else {
        Button { showPaywall = true } label: {
            HStack {
                Image(systemName: icon).foregroundStyle(.secondary).frame(width: 24)
                Text(title).foregroundStyle(.primary)
                Spacer()
                Image(systemName: "lock.fill").font(.caption2).foregroundStyle(.yellow)
            }
        }
    }
}
```

**应用 Pro 拦截**
